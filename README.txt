# README - Ανάλυση Σχεδιαστικών Επιλογών και Υλοποίησης

## Εισαγωγή

Αυτή η εργασία υλοποιεί ένα σύστημα ηλεκτρονικού καταστήματος (e-shop) που υποστηρίζει δύο τύπους χρηστών: 
**Διαχειριστές (Admins)** και **Πελάτες (Customers)**. Στόχος της λύσης ήταν η δημιουργία μιας 
εφαρμογής που καλύπτει βασικές λειτουργίες διαχείρισης και αγοράς προϊόντων.

---

## Σχεδιαστικές Επιλογές

### 1. Αντικειμενοστραφής Προσέγγιση
Η υλοποίηση βασίζεται στην αντικειμενοστραφή προσέγγιση με τη χρήση κλάσεων, ώστε να επιτυγχάνεται:
- **Επαναχρησιμοποίηση Κώδικα**: Οι κοινές λειτουργίες ενσωματώνονται στην κλάση `User`, ενώ οι εξειδικευμένες λειτουργίες υλοποιούνται στις υποκλάσεις `Admin` και `Customer`.
- **Επεκτασιμότητα**: Το σύστημα μπορεί να επεκταθεί εύκολα με νέες λειτουργίες ή τύπους χρηστών.
- **Ευκρίνεια Κώδικα**: Ο διαχωρισμός ευθυνών ανά κλάση βελτιώνει τη συντηρησιμότητα.

### 2. Αρχειοθετημένη Αποθήκευση Δεδομένων
Τα δεδομένα αποθηκεύονται σε αρχεία κειμένου (`categories.txt`, `products.txt`, `users.txt`), διευκολύνοντας την πρόσβαση και την ενημέρωσή τους χωρίς τη χρήση βάσης δεδομένων.

- **Αρχείο `categories.txt`**: Περιλαμβάνει κατηγορίες και υποκατηγορίες προϊόντων.
- **Αρχείο `products.txt`**: Αποθηκεύει πληροφορίες για τα προϊόντα, όπως τίτλος, περιγραφή, κατηγορία, υποκατηγορία, τιμή και ποσότητα.
- **Αρχείο `users.txt`**: Περιέχει τους εγγεγραμμένους χρήστες και τον τύπο τους (Admin/Customer).
- **Φάκελος `order_history`**: Κρατά το ιστορικό παραγγελιών κάθε πελάτη ξεχωριστά.

### 3. Λειτουργίες Χρηστών
Οι λειτουργίες διαχωρίζονται ανάλογα με τον ρόλο του χρήστη:
- **Διαχειριστής (Admin)**:
    - Προσθήκη Προϊόντων: Οι διαχειριστές μπορούν να προσθέσουν νέα προϊόντα με πληροφορίες όπως τίτλος, περιγραφή, κατηγορία, υποκατηγορία, τιμή και ποσότητα.
    - Επεξεργασία Προϊόντων: Οι διαχειριστές μπορούν να τροποποιήσουν πληροφορίες για υπάρχοντα προϊόντα.
    - Διαγραφή Προϊόντων: Δυνατότητα διαγραφής προϊόντων από το κατάστημα.
    - Αναζήτηση Προϊόντων: Οι διαχειριστές μπορούν να αναζητήσουν προϊόντα βάσει τίτλου, κατηγορίας ή κατάστασης διαθεσιμότητας.
    - Προβολή Μη Διαθέσιμων Προϊόντων: Προβολή προϊόντων με μηδενική διαθεσιμότητα.
    - Προβολή Top 5 Προϊόντων: Τα προϊόντα που έχουν εμφανιστεί στις περισσότερες παραγγελίες.
- **Πελάτης (Customer)**:
    - Αναζήτηση Προϊόντων: Οι πελάτες μπορούν να αναζητήσουν προϊόντα βάσει τίτλου ή κατηγορίας.
    - Προσθήκη Προϊόντος: Οι πελάτες μπορούν να προσθέσουν προϊόντα στο καλάθι τους.
    - Ενημέρωση Καλαθιού: Οι πελάτες μπορούν να ενημερώσουν την ποσότητα των προϊόντων που έχουν στο καλάθι τους.
    - Αφαίρεση Προϊόντος: Οι πελάτες μπορούν να αφαιρέσουν ήδη υπάρχοντα προϊόντα από  το καλάθι τους.
    - Ολοκλήρωση Παραγγελίας: Υπολογισμός κόστους παραγγελίας και αποθήκευση στο ιστορικό παραγγελιών.
    - Ιστορικό Παραγγελιών: Προβολή παλαιότερων παραγγελιών.
    - Εμφάνιση Καλαθιού: Προβολή του τρέχοντος καλαθιού.

### 4. Δομή Κώδικα

- **oop24.cpp**: Το κύριο αρχείο της εφαρμογής που ξεκινά τη ροή του προγράμματος.
- **Admin.h / Admin.cpp**: Υλοποιεί τις λειτουργίες διαχειριστή.
- **Customer.h / Customer.cpp**: Υλοποιεί τις λειτουργίες πελάτη.
- **User.h / User.cpp**: Βασική κλάση χρήστη, κληρονομούμενη από Admin και Customer.
- **Product.h / Product.cpp**: Υλοποιεί την κλάση προϊόντος.
- **Cart.h / Cart.cpp**: Υλοποιεί το καλάθι αγορών.
- **Utility.h / Utility.cpp**: Παρέχει βοηθητικές συναρτήσεις, όπως η φόρτωση δεδομένων από αρχεία.
